Note — this targets the Cardputer ADV on the Stamp-S3A core, not the original Cardputer. The ADV adds a larger battery, a BMI270 IMU, ES8311 audio, an IR emitter, and a microSD slot. Those peripherals are present on the board but not yet wired up by this firmware — they're free for you to use.
02

Features

Launcher menu

Five built-in apps, one-handed up/down browsing. OPT is a universal back key from anywhere.

Live status bar

Current app title plus a battery gauge — percentage and colour state, refreshed roughly every 2 seconds.

NSH terminal

Wired to a ring-buffer character device with 200 lines of scrollback history you can page through.

VT100 handling

Cursor moves, erase line/screen, and colour SGR sequences are parsed and consumed for the shell.

Fast row flushes

Printable characters flush a single dirty row for snappy typing; full repaints only on newline or scroll.

Vector graphics

Rounded rects, gradients, and icons drawn in code. No external image assets to ship or load.

03

Built-in apps

ScreenTest01
Cycles full-screen solid colours to check for dead pixels or tint. A centered card names each colour and its index.
Keyboard02
Live key inspector — decoded character, raw scan code, and the SH / CT / FN modifier state. Great for verifying the matrix.
Terminal03
Interactive NSH shell on the LCD with command history and 200-line scrollback. Bottom-anchored; output fills upward.
Files04
Simple browser starting at root. ENTER opens directories, .. goes up, and a counter shows position in long listings.
Battery05
Large percentage gauge, raw millivolts, and a Full / OK / Low status. Degrades to a diagnostic if the ADC is unreadable.

Build & flash

~/nuttx — standard ESP32-S3 build flow
# place esp32s3_bringup.c in the board source, then configure
cd nuttx && ./tools/configure.sh esp32s3-devkit:nsh
make menuconfig # enable LCD / ST7789 / ADC / SPIFLASH
# build → produces nuttx.bin + bootloader + partition table
make -j
CC: esp32s3_bringup.c
LD: nuttx
BIN: nuttx.bin
# flash over USB-C (side switch ON)
make flash ESPTOOL_PORT=/dev/ttyACM0 ESPTOOL_BAUD=921600
Writing... done. Hash of data verified.
make monitor ESPTOOL_PORT=/dev/ttyACM0 # optional console
Need download mode? Hold G0 while tapping reset. On the ESP32-S3 native USB auto-reset usually makes this unnecessary.

The keyboard has an fn modifier for the arrow layer, plus shift, ctrl, and OPT as a universal back. In menus and browsers, i/k also move up/down one-handed; in the terminal, arrows always need fn so typing isn't intercepted.
